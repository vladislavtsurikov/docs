---
title: "Brush Erase"
description: "Erase objects and terrain details with a brush using filter-based and per-prototype erase control"
---

Brush Erase is the erasing counterpart to the painting workflows. It lets you remove placed objects or terrain details directly in the scene with a brush.

Instead of erasing everything uniformly, the tool combines brush strength, filter fitness, mask influence, and per-prototype erase success.

## Supported resource types

Brush Erase supports:

- **Terrain Object**
- **Unity/GameObject**
- **Unity/Terrain Detail**

## Core workflow

1. Select **Brush Erase** in the Tools panel.
2. Choose the groups and prototypes you want to erase from.
3. Set brush size and spacing.
4. Adjust erase strength and per-prototype erase success.
5. Erase directly in the scene.

## Tool-level controls

Brush Erase uses:

- **Brush Erase Tool Settings**
- **Brush Settings**

The main tool-specific control is:

- **Erase Strength** which scales how strongly the erase pass removes data.

## Shared systems used by this tool

For **Terrain Object** and **Unity/GameObject** groups, Brush Erase uses:

- [Filter Settings](/megaworld/mask-filters/overview)
- `Additional Erase Settings` on each prototype

For **Unity/Terrain Detail** groups, Brush Erase uses:

- mask filter settings
- `Additional Erase Settings` on each prototype

## How erase is evaluated

Brush Erase does not simply delete everything under the brush. It combines:

- the current brush mask;
- filter fitness or mask-filter result;
- **Erase Strength**;
- the per-prototype erase success value.

This makes it possible to erase softly, selectively, or with variation.

## When to use Brush Erase

Brush Erase is best when you want:

- to clean up unwanted placements manually;
- to thin out dense areas without deleting everything;
- to erase terrain details with mask-aware behavior;
- an erasing workflow that still respects MegaWorld’s filtering logic.

<Tip>
  If erasing feels too aggressive, reduce **Erase Strength** first. If only certain prototypes should survive more often, lower their per-prototype erase success instead of changing the whole tool.
</Tip>
