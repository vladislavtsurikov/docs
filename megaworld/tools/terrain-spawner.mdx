---
title: "Terrain Spawner"
description: "Spawn objects and terrain details inside a bounded area using filters, scatter, and prototype settings"
---

Terrain Spawner is a stamper tool for procedural placement inside a defined area. You position and scale the stamper in the scene, then MegaWorld evaluates the selected groups and fills that volume with the prototypes that pass your rules.

This is one of the best tools when you want to generate a large area at once instead of painting manually.

## Features

### Spawn large terrain objects in one click

Terrain Spawner fills the entire stamper area in a single pass. For very large terrains, enable **Use Spawn Cells** so the area is processed cell by cell instead of all at once. This avoids long freezes and prevents Unity from crashing on extremely large areas.

### Spawn support for all resource types

Terrain Spawner works with every MegaWorld resource kind in a single tool:

- **Terrain Object** and **Unity/GameObject** — procedural object placement with the full shared stack
- **Unity/Terrain Detail** — grass and mesh detail density passes
- **Unity/Terrain Texture** — terrain texture painting via mask filters

### Shared settings across tools

Types in MegaWorld are reusable presets. The same Type you configure in Terrain Spawner can be dragged directly into Advanced Brush and used immediately. This means you can build and tune a preset in the stamper using Auto Respawn, then hand-paint with the same rules in the brush — no duplication needed.

### Auto Respawn

When **Enable Auto Spawn** is on, Terrain Spawner automatically removes and re-spawns the current pass every time you change a setting. Use this during setup to get instant feedback, especially for terrain texture passes which run on the GPU and can be applied quickly even at large scales. Set **Delay Auto Spawn** to control the respawn delay in seconds.

### Works on Unity Terrain and terrain mesh

Terrain Spawner works on both Unity Terrain and polygon terrain meshes such as [Polaris](/megaworld/polaris-integration). For mesh terrain, switch to **Simple Filter** instead of Mask Filters — Simple Filter supports height, slope, and noise placement rules on any surface.

## Supported resource types

Terrain Spawner supports:

- **Terrain Object**
- **Unity/GameObject**
- **Unity/Terrain Detail**

## Core workflow

1. Add a Terrain Spawner to the scene.
2. Position and scale the stamper area.
3. Select the groups and prototypes you want to use.
4. Configure filters, scatter, transform, overlap, and prototype-specific settings.
5. Run the spawn pass inside the stamper bounds.

## Shared systems used by this tool

For **Terrain Object** and **Unity/GameObject** groups, Terrain Spawner uses:

- `Random Seed Settings`
- scatter settings
- [Filter Settings](/megaworld/mask-filters/overview)
- [Transform Components](/megaworld/transform-components/overview)
- [Overlap Check Settings](/megaworld/overlap-check-settings/overview)
- [Additional Spawn Settings](/megaworld/general-settings/additional-spawn-settings)

For **Unity/Terrain Detail** groups, Terrain Spawner uses:

- [Spawn Detail Settings](/megaworld/general-settings/spawn-detail-settings)
- mask filter settings

## Tool-level controls

### Area Settings

Area Settings define the size and position of the stamper volume in the scene. Key controls:

- **Fit To Terrain Size** — resizes the area to cover the entire Unity Terrain in one click.
- **Use Spawn Cells** — splits the area into a grid of cells that are spawned independently. Recommended for very large terrains to avoid freezes or crashes.
- **Handle Settings** — control how the area box is drawn in the scene view (color, line width, dotted lines, visibility when deselected).

See the full [Area Settings](/megaworld/shared-settings/area-settings) reference for all options.

### Stamper Tool Controller

The Stamper Tool Controller is the main action and mode panel for the Terrain Spawner.

<img
  style={{ borderRadius: '0.5rem', width: '100%', maxWidth: '960px' }}
  src="/images/megaworld/tools/terrain-spawner/stamper-tool-controller.png"
  alt="Stamper Tool Controller settings."
/>

**Enable Visualization** — shows the Mask Filter output directly in the scene view. Lets you see exactly which areas pass your height, slope, and noise filters before spawning. Visualization appearance is configured in [Preferences](/megaworld/preferences/overview).

**Enable Auto Spawn** — automatically removes and re-spawns the current pass every time you change a setting. This gives you instant live feedback while tuning filters and scatter rules. See [Auto Respawn](#auto-respawn) feature for details.

- **Delay Auto Spawn** — how many seconds to wait after a setting change before triggering the automatic respawn. Increase this if your area is large and you don't want a respawn on every small adjustment.

**Respawn** — removes all currently spawned instances from this stamper and runs a fresh spawn pass. Use this to regenerate the result after making manual changes.

**Spawn** — adds a new spawn pass on top of whatever is already in the scene. Each click of Spawn adds more instances without clearing the previous result.

**Unspawn Selected Prototypes** — removes only the instances that belong to the currently selected prototypes, leaving everything else untouched. Useful for clearing a specific resource type (e.g., only the grass layer) without redoing the whole stamper.

## When to use Terrain Spawner

Terrain Spawner is best when you want:

- a one-shot generation pass over a large area;
- a forest, rock field, or vegetation layout based on filters and scatter rules;
- terrain detail spawning inside a selected zone;
- a workflow where the result is tied to a scene stamper object instead of mouse painting.

## Notes

- Terrain Spawner evaluates each selected group separately inside the stamper bounds.
- For object-based groups, the tool generates random seeds when needed before sampling scatter.
- For terrain details, the workflow is more specialized and uses terrain-detail-specific settings instead of the standard success and transform stack.

<Tip>
  Start with filters and scatter first, then add transform components and overlap rules. That order makes Terrain Spawner much easier to tune.
</Tip>
